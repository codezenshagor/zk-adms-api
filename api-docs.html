API Usage Guide

ZK device এ user add, delete, fingerprint/face enroll, door control এবং attendance data fetch করার জন্য complete API reference। Postman, PHP এবং Node.js example সহ।

Base URL: http://ser.test REST / JSON API Key Auth Postman PHP / Node.js
AUTH Authentication

প্রতিটি request এ এই header দুটি mandatory।

Request Headers 
X-API-KEY: YOUR_CLIENT_API_KEY
Content-Type: application/json
📌 cmd_id rule: প্রতিটি request এ unique cmd_id দিতে হবে। Recommended format: timestamp + sequence → 170000000001। Duplicate cmd_id reject হবে।
01 Postman Setup

Postman দিয়ে API test করার step-by-step guide।

1
Collection তৈরি করুন
Postman → New Collection → Name: "ZK Push Command API"
2
Environment Variable সেট করুন
Environments → New → Add variables: base_url = http://ser.test এবং api_key = YOUR_CLIENT_API_KEY
3
Collection-level Auth সেট করুন
Collection → Edit → Authorization → Type: API Key → Key: X-API-KEY → Value: {{api_key}} → Add to: Header
4
Request তৈরি করুন
New Request → Method: POST → URL: {{base_url}}/api/command/send → Body: raw → JSON → নিচের example body paste করুন।
5
Pre-request Script (auto cmd_id)
Tests tab এ এই script দিন — auto unique cmd_id generate হবে:
Postman Pre-request Script 
// Pre-request Script → auto-generate unique cmd_id
const ts  = Date.now();
const seq = Math.floor(Math.random() * 1000).toString().padStart(3, '0');
pm.environment.set('cmd_id', ts + seq);
Body তে "cmd_id": "{{cmd_id}}" ব্যবহার করলে auto unique ID পাবেন।

Example Calls
cURL — Attendance
# Today's all attendance
curl -G http://ser.test/api/attendance \
  -H "X-API-KEY: YOUR_CLIENT_API_KEY" \
  --data-urlencode "date=2026-03-31"

# Date range for one employee
curl -G http://ser.test/api/attendance \
  -H "X-API-KEY: YOUR_CLIENT_API_KEY" \
  --data-urlencode "from=2026-03-01" \
  --data-urlencode "to=2026-03-31" \
  --data-urlencode "pin=1001"
<?php
// Today attendance
$params = http_build_query(['date' => date('Y-m-d')]);
$ch = curl_init("http://ser.test/api/attendance?{$params}");
curl_setopt_array($ch, [
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_HTTPHEADER     => ['X-API-KEY: YOUR_CLIENT_API_KEY']
]);
$data = json_decode(curl_exec($ch), true);
curl_close($ch);

foreach ($data['records'] as $log) {
  echo "{$log['main_id']} | {$log['timestamp']} | state:{$log['state']}\n";
}

# ZK Attendance Push API

ZK device এর attendance data receive করে DB তে store করার জন্য PHP-based push receiver এবং device management command API।

## Features
- ✅ ZK PushDispatcher থেকে attendance data receive
- ✅ Multiple device support (serial + API key per device)
- ✅ SQLite এবং MySQL উভয় support
- ✅ User add/delete/bulk operations
- ✅ Fingerprint এবং Face enrollment
- ✅ Door control (open/lock/unlock)
- ✅ Command status tracking

## Requirements
- PHP 8.0+ (PDO, SQLite/MySQL extension)
- Web server (Apache/Nginx)
- ZK device with PushDispatcher enabled

## Quick Setup

### 1. Device Configuration
Device এর Server Settings এ দিন:
- Server Address: `https://yourdomain.com`
- Server Port: `443`
- Push Path: `/zkrcv2`
- Push Interval: `30` (seconds)

### 2. Database Setup

**SQLite:**
```php
$pdo = new PDO('sqlite:' . __DIR__ . '/attendance.db');
$pdo->exec("PRAGMA journal_mode=WAL");
```

**MySQL:**
```sql
CREATE TABLE att_log (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  main_id VARCHAR(50) NOT NULL,
  state VARCHAR(5) DEFAULT '0',
  timestamp DATETIME NOT NULL,
  type VARCHAR(20) DEFAULT 'attlog',
  device_serial VARCHAR(100) DEFAULT NULL,
  UNIQUE KEY uq_punch (main_id, timestamp, device_serial)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3. API Key Setup
```php
define('EXPECTED_API_KEY', 'your-secret-api-key-here');
```

## API Usage

### Add User
```bash
curl -X POST http://yourdomain.com/api/command/send \
  -H "X-API-KEY: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"cmd_id":"170000000001","sn":"DEVICE_SN","cmd_type":"set_user","pin":"1001","name":"Rahim","role":0}'
```

### Get Attendance
```bash
curl -G http://yourdomain.com/api/attendance \
  -H "X-API-KEY: YOUR_KEY" \
  --data-urlencode "date=2026-03-31"
```
